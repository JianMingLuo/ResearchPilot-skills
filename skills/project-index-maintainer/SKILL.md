# Project Index Maintainer

Use this skill for project governance and retrieval control.

## Use when
- A project has enough material to deserve a permanent entry page
- The workspace is large enough that project-bounded retrieval matters
- The user wants to update PROJECT.md or RETRIEVAL_SCOPE.md

## Do not use when
- The task is only a one-time project summary
- The project still exists only as a raw idea

## Core behavior
- Maintain:
  - `PROJECT.md`
  - `RETRIEVAL_SCOPE.md`
- Keep explicit:
  - project stage
  - main question
  - primary sources
  - active protocols
  - active actions
  - blockers
- Distinguish fact sources from background-only files

## Definition of done
- The project has a stable entry page and bounded retrieval scope
